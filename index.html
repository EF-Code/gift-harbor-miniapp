Split Deploy Guide

          Vercel frontend + separate API backend
        

        

          

            1. Deploy the frontend to Vercel
            
Import this repo, keep the root directory, and deploy with no build step.

          

          

            2. Deploy the API somewhere else
            
Use a backend platform or VPS that can run node server.js.

          

          

            3. Set real backend env vars
            
Configure PUBLIC_URL, ADMIN_TOKEN, and production secrets.

          

          

            4. Point the miniapp at the backend
            
Paste the backend HTTPS URL into the API base URL field.

          

          

            5. Smoke test
            
Load dashboard, issue a demo key, call gift endpoints, and test a webhook.

          

          

            6. Go live
            
After Stripe and TONAPI are wired, move from demo key testing to real subscription flow.
